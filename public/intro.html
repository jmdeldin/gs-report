Jon-Michael's Feedback

Cosmetic changes

  • Don't connect the user to the destination in the map - use an indicator/compass so they don't have to go straight to the class
    • This was a great idea because it gives the user the flexibility to run a few errands before their class. The user will not feel like they are forced to go to class. This has been incorporated.
  • Have the usual "Edit" and "+" top nav in the app.
    • This will enable navigation between screens and has been incorporated.

Changes to functionality

  • Add classes that are not in catalog
    • Most users will have classes that are in the catalog, and if they don't, they can lookup a building/room instead. However, we do not plan to incorporate custom class creation.
  • Integrate with the Academic Planner
    • Unfortunately, getting access to the academic planner means dealing with campus IT red-tape. We are not incorporating this for now, but perhaps a v3.0 could integrate.
  • Show class times and locations
    • We are incorporating this into the class schedule screen.
  • Allow searching by class abbreviation (e.g., CSCI instead of just "Computer Science")
    • We have chosen to adopt this as the sole method of finding classes, so instead of searching by class name, one searches by class number after selecting a department (e.g., "ECON 511").
  • Add a map tab that shows saved classes as pins.
    • We are incorporating this as the main screen of the app (after the brief splash screen). Users will have the option to view all classes at once or view one class at a time.

Changes to UI widgets and controls

  • Remove the "Save" button from the search results/map view
    • In the consolidation process, we have front-loaded this step - users decide to save before they view the class on a map.
  • Remove "Undo" button after saving a class, since it isn't really needed and is confusing
    • We have incorporated this - users can remove a class by viewing their class list and then deleting.
  • Have two modes for searching for classes. The "slow" one would be a dropdown to filter by subject, by class, and by section. The "fast" one would be a search box to autocomplete.
    • We have incorporated this by showing a list of classes with a search/filter box at the top.
  • Have a way to cancel/clear out search (like the little "X" in Safari's location bar)
    • We have incorporated this on the search/filter text boxes.

Changes to task flow

  • Include a map on the front page, instead of defaulting to "find a class"
    • We have incorporated this in order to minimize the amount of clicks/taps required to view directions.

Functional Requirements

High Priority - Core Functionality

User should be able to:

  1. view full map of campus with his/her current location indicated
    1. pan/zoom
  2. clear the full map of any annotations
  3. search for a course
  4. add a course to his/her course list
  5. view an annotated map with a selected course pinned and his/her current location pinned
  6. see directions to a course location from current location
  7. view all of his/her classes on the full map
  8. view course details
  9. remove course from course list
  10. find location of his or her "next class"
  11. search for points of interest and see their selection on a map in relation to their current location
  12. see directions to a selected point of interest
  13. be alerted when they have arrived at their selected point of interest
  14. be alerted of an upcoming course time (their next class)

Medium Priority Functionality

  1. "I'm Bored" feature

    Users should be able to:

    1. configure activities that they are interested in based on a pre-determined list
    2. view randomly-selected activities based on pre-configured interests
    3. view the location of the randomly-selected activities on the full campus map with his/her location indicated
    4. view directions to the randomly-selected activity
  2. Griz Photos feature

    Users should be able to:

    1. view photos of campus pulled from Flickr based on proximity to current location
  3. View floorplans

    Users should be able to:

    1. view a floorplan of the building that he/she is viewing on the map after either "finding" it or after finding a course or after arriving at a building.
  4. Find a Study Buddy

    Users should be:

    1. able to indicate that they are interested in finding a classmate(s) to study with based on the courses that they are in.
    2. alerted when another user has indicated he/she is interested in finding a study buddy for the same course.
    3. able to send an email to a potential study buddy to initiate contact.

Low Priority Functionality

  1. Website
    1. Users should be able to view contact information, FAQs, troubleshooting help, etc.
    2. Users should be able to file support tickets and suggestions
  2. Users should be able to
    1. view a course schedule in calendar format
    2. save a waypoint
    3. view list of saved waypoints and view directions to
    4. view floorplan for a selected building
  3. Social
    1. Users should be able to select interests from a pre-configured list, such as racquetball, eating, etc
    2. Users should be able to view other users with similar interests on the full campus map based on their location
    3. Users should be able to initiate contact with other users with similar interests
